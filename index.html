KLayout + Python + MCP

klink

面向 KLayout 的 AI-native 控制平面。它把正在运行的 KLayout GUI 变成可由外部 Python、MCP client 和 AI agent 控制的版图内核——geometry-first：在几何上标意图，让算法补全布线，用结构化几何和 LVS 判定完成，而不是靠“看起来像”的截图。这是一个早期试验阶段的开源项目，欢迎试用和反馈。

Package: klayout-klink Python 3.10+ KLayout 0.30.x tested Apache-2.0 开源 Alpha · 早期阶段

不是普通聊天插件

klink 的核心方法是 geometry-first：几何在先，你在几何上标注意图（Port 连接、Anchor 路由约束），routing 算法据此补全布线，再用结构化查询和 live LVS 判定完成。人保留 GUI 自由，agent 读、写、搬运、录制、路由和验证真实几何。

Concept

geometry-first 思路

标记意图 → 算法布线 → 结构化验证。Port/Anchor 是路由后端的输入，机制层保持工艺无关。

安装 Python 包

PyPI 包带 klink 核心和自带的两个 Rust 加速内核；starter demo 由 klink init 复制进用户项目，升级后用 klink update 刷新（不碰你自己的文件）。

安装 KLayout 插件

把 klink_plugin/ 复制到 KLayout 的 salt 目录并重启。插件起 8765 RPC 和 8082 klive 显示。

连接 agent

运行 python -m klink.mcp，用 profile 控制 read/write/verify/escape 或指定领域工具。

控制路径

薄插件贴着 KLayout，重的智能在外部 Python。两侧只经 RPC 通信。细节见 架构。

control path
Agent (Claude Code / Codex)
  → MCP server   (python -m klink.mcp)
  → klink client (NDJSON over TCP)
  → KLayout plugin (in-process RPC)
       RPC:            127.0.0.1:8765+
       klive-compat:   127.0.0.1:8082

会话即端口，平等对等

每个 KLayout 窗口绑定一个端口（8765、8766…）注册为一个 session。没有特权端口，显式指明目标。

Process purity

klink 不带工艺常量。层号、尺寸、器件都由你的 pdk.py 显式传入；缺参数返回带 next_action 的指导性错误。
